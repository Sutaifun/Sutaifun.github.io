第六次实验问题汇总

A.电力分配

这道题目在强调了一一对应之后，不难发现这是一道贪心算法的题目（题目甚至明示了贪心算法）。
思路很简单：对于所需功率最小的生产线，总是尝试使用提供功率尽可能小的发电机进行供电；
或者对于提供功率最大的发电机，总是尝试对所需功率尽可能大的生产线进行供电。
这两个思路虽然最终分配方案不一定一样，但是可供电的生产线数量是相等的。

正确答案

n = int(input())
lnum = []
for _ in range(n):
    lnum.append(int(input()))
lnum.sort()                                 # 注意题目并没有说输入时满足某种次序
m = int(input())                            # 所以需要自行.sort()进行排序
ln2 = []
for _ in range(m):
    ln2.append(int(input()))
ln2.sort()
ans = 0
for i in ln2:                               # 使用上述第一种思路
    if lnum[0] <= i:
        ans += 1
        lnum.pop(0)
        if lnum == []:                      # 进行特判避免索引越界
            break
print(ans)

B.我不是盘神

这道题目不难想到先用XXXDownload下载两个最大的文件，然后再用XX网盘下载一个最小的文件，如此循环。
本质贪心算法，但是不用去模拟这一过程。我们只需要算出哪些文件需要用XX网盘下载即可。

正确答案

不难算出，需要用XX网盘下载的文件是从小到大排序后前n//3个文件。 

n = int(input())
lnum = [int(input()) for i in range(n)]     # 列表生成式输入数据
lnum.sort()                                 # 排序
lans = lnum[:n//3]                          # 切片操作取出需要用XX网盘下载的文件
print(sum(lans))                            # 输出总大小
print(' '.join(list(map(str, lans))))       # 依次输出文件大小

C.装备选择

通过仔细阅读题目，这道题目就是换了马甲的背包问题。所以这里不做重复的解释了。
如果还是有问题，请查阅课件里相应的讲解。

正确答案

r = int(input())
leq = [0]
for _ in range(r):
    leq.append(tuple(map(int, input().split())))
n = int(input())                            # 关于下面value的生成需要注意一下
value = [[0 for i in range(n+1)] for j in range(r+1)]
for i in range(1, r+1):
    for j in range(1, n+1):                 # 状态转移方程，为了避免越界需要分支
        if j >= leq[i][0]:
            value[i][j] = max(value[i-1][j], value[i-1][j-leq[i][0]]+leq[i][1])
        else:
            value[i][j] = value[i-1][j]
print(value[-1][-1])                        # 状态表格最右下即使答案

cnt = 0                                     # 计数器
ans = []                                    # 存储答案
mon = n
for i in range(r, 0, -1):
    if value[i][mon] != value[i-1][mon]:    # 不等于说明选取了当前装备
        cnt += 1                            # 计数器加一
        ans.append(str(i-1))
        mon -= leq[i][0]                    # 注意这一步，要减少相应的金币数
print(cnt)
print(' '.join(ans))

D.我不是盘神Pro（选做）

这道题目相比B题，多了一个限制条件，即不能改变下载的顺序，所以原先的下载策略不再适用于这道题目。
可以采用动态规划完成这道题目。
但是尝试写出状态转移方程时，发现影响状态的参数不仅仅是已下载的文件数量。

正确答案

使用一个n*3的列表存储状态，其中l[n][k]指下载n个文件且目前已经连续使用XXXDownload下载了k个文件时的最优解。
l[n][0]是指第n个文件使用XX网盘下载，需要取l[n-1][k]的最小值加上第n个文件的大小；
l[n][1]值第n个文件使用XXXDownload下载，第n个文件使用XX网盘下载，所以只需要取l[n-1][0]的值即可；
l[n][2]值第n个文件和第n-1个文件使用XXXDownload下载，第n-2个文件使用XX网盘下载，所以只需要取l[n-1][1]的值即可。

n = int(input())
lnum = [int(input()) for i in range(n)]
lcon = [[0 for i in range(3)] for j in range(n)]
for i in range(n):                          # 这里无需进行特判，i=0时，i-1为-1
    lcon[i][0] = min(lcon[i-1][0], lcon[i-1][1], lcon[i-1][2]) + lnum[i]
    lcon[i][1] = lcon[i-1][0]               # 开始时lcon[-1][k]均为0，不会影响最终结果
    lcon[i][2] = lcon[i-1][1]
print(min(lcon[-1]))                        # 输出最小的情况

优化解法

上述解法还有优化空间。因为lcon[i][1] = lcon[i-1][0]，lcon[i][2] = lcon[i-1][1] = lcon[i-2][0]，我们发现完全没有必要使用后两列数据，所以可以把存储状态的列表压缩成一维。
此时lcon[i]表示使用XX网盘下载第n个文件时所耗总时间。 

n = int(input())
lnum = [int(input()) for i in range(n)]
lcon = [0 for j in range(n)]
for i in range(n):
    lcon[i] = min(lcon[i-3], lcon[i-2], lcon[i-1]) + lnum[i]
print(min(lcon[-3:]))

列表复制

如果我们想把一个列表赋给另一个变量名，并且希望在更改新的列表时不会影响之前的列表，那么我们可以使用列表的.copy()方法创建列表的复制。 

>>> a = []
>>> b = a.copy()
>>> b.append(1)
>>> b
[1]
>>> a
[]

为什么我在创建n的时候明明使用了.copy()，却还是在改变n的时候同时改变了l和a？
我们还是来看一下最后的状态下数据存储的方式：

虽然n确实是复制了l里的所有元素，但是l[0]这一项的值是“指向某一个list”，所以l[0]和n[0]还是指向了同一个list。
但是修改n[1]的时候，l[1]并没有一起改变，我们可以将.copy()理解为“只进行了一层复制”。这种复制称为浅复制。
如果我想创造一个列表，与之前的列表完全一样，但是完全独立，二者互不影响，该怎么办呢？
这时我们需要引入copy库来完成这样的操作。
（接上述交互式窗口）

>>> import copy
>>> s = copy.deepcopy(l)
>>> s
[[0, 0, 0, 1, 0], 1]
>>> s[0][2] = 3
>>> s
[[0, 0, 3, 1, 0], 1]
>>> l
[[0, 0, 0, 1, 0], 1]
>>> a
[0, 0, 0, 1, 0]

这时，我们发现改变s的值也不会引起其他变量的变化。
